# White Block Fix - What Changed

## TL;DR

**Problem:** White block visible before splash screen (both TFT and LVGL)  
**Root Cause:** Missing backlight gradient fade during initialization  
**Solution:** Added LilyGo-recommended gradient fade (0→255, 2ms per step)

---

## Files Modified

### 1. `src/display/display_core.cpp` (TFT Path)
- **What:** Added backlight gradient fade during initialization
- **Location:** `init_display()` function
- **Code:**
  ```cpp
  for (uint8_t brightness = 0; brightness < 255; brightness++) {
      ledcWrite(0, brightness);
      smart_delay(2);  // Critical timing
  }
  ```
- **Effect:** TFT path now fades backlight smoothly

---

### 2. `src/display/display_splash.cpp` (TFT Splash)
- **What:** Simplified splash sequence, removed redundant backlight control
- **Location:** `displaySplashWithFade()` function
- **Changes:**
  - Removed "Backlight OFF" step at start
  - Changed from 5-step to 3-step sequence
  - Cleaner logging
- **Effect:** No sudden backlight on/off transitions

---

### 3. `src/hal/display/lvgl_driver.cpp` (LVGL Path) ⭐ **NEW**
- **What:** Added backlight gradient fade to LVGL initialization
- **Location:** `LvglDriver::init()` function, STEP 5
- **Code:**
  ```cpp
  for (uint8_t brightness = 0; brightness < 255; brightness++) {
      ledcWrite(HardwareConfig::GPIO_BACKLIGHT, brightness);
      smart_delay(2);  // Critical timing
  }
  current_backlight_ = 255;
  ```
- **Effect:** LVGL path now fades backlight smoothly during init

---

## Why Both Files Needed Fixing

Your system supports **two display backends**:

| Backend | File | Status |
|---------|------|--------|
| TFT | display_core.cpp | ✅ Fixed |
| LVGL | lvgl_driver.cpp | ✅ **Just Fixed** |

Both had the same problem: missing hardware backlight fade.

---

## The Critical Detail

### What LVGL Code Was Doing (Wrong)
```cpp
// In display_splash_lvgl.cpp
set_backlight(0);  // Keep at 0
// Later: LVGL opacity animation fades the image
lv_anim_set_values(&anim_in, 0, 255);  // Opacity fade, not backlight!
```

**Problem:** This fades the LVGL image opacity, not the hardware backlight.  
Backlight stays at 0 until the animation happens, causing the white block.

### What We Fixed (Correct)
```cpp
// In lvgl_driver.cpp STEP 5
for (uint8_t brightness = 0; brightness < 255; brightness++) {
    ledcWrite(HardwareConfig::GPIO_BACKLIGHT, brightness);  // Hardware fade
    smart_delay(2);  // Panel settling
}
current_backlight_ = 255;
```

**Solution:** Hardware backlight fades during init, before any LVGL content.

## Backlight Fade Specifications

| Parameter | Value |
|-----------|-------|
| Start Level | 0 |
| End Level | 255 |
| Number of Steps | 255 |
| Delay Per Step | 2ms |
| Total Fade Time | ~510ms |
| PWM Frequency | 10kHz |
| PWM Resolution | 8-bit |
| Timing Source | LilyGo reference code |

## Boot Sequence Comparison

### Before (Broken)
```
1. Power ON
2. Display HW init (backlight OFF)
3. LVGL init
4. Render black bootstrap
5. Backlight suddenly ON (0→255 instantly)
6. ❌ White block visible
7. Splash loads
```

### After (Fixed)
```
1. Power ON
2. Display HW init (backlight OFF)
3. LVGL init
4. Render black bootstrap
5. Backlight GRADIENT FADE (0→255, 2ms steps)
6. ✅ Smooth fade-in, no white block
7. Splash loads
```
